## Ответы на "философские" вопросы

1. **Зачем нужны Java и Docker?**
    * **Java:** Чтобы один раз написать код и он работал везде, где есть JVM (и чтобы чувствовать боль и величие строгого типизирования).
    * **Docker:** Чтобы фраза "на моей машине всё работает" больше не была оправданием. Упаковываем зависимости в контейнер и запускаем где угодно без "танцев с бубном".

2. **Почему вы захотели быть программистом?**
    * Чтобы работать ml-щиком, писать модельки и зарабатывать много денюжек.

3. **Почему я ненавижу Windows и продукты Microsoft?**
    * Потому что Windows — это реестр, обновления в самый неподходящий момент, отсутствие нормального пакетного менеджера (до недавнего времени) и вечные проблемы с путями.

4. **Какие интересы у вашего любимого преподавателя?**
    * Требовать аниме-картинки и слушать Amatory.

## Идея приложения: "Guitar Shop"
Магазин инструментов, где можно посмотреть каталог гитар. Но есть одна специально скрытая модель.

### Архитектура (Block Scheme)


1. **Frontend (Static/JS):** Простая "кликалка" на HTML/JS. Показывает товары.
2. **Backend (Spring Boot):** REST API на Java. Обрабатывает запросы, фильтрует "видимые" товары.
3. **Database (H2/DB):** Хранилище гитар. Содержит скрытые записи (visible: false).
4. **Spy Terminal:** Отдельная страница, которая через CORS-уязвимость (эксплойт) вытягивает секретные данные напрямую из API.

---

## Spy & CORS Feature
В приложении реализован механизм **CORS**.
* Основная страница видит только разрешенные товары.
* **Шпионская страница (`spy.html`)** имитирует атаку и делает запрос к API, доставая скрытые записи (например, `SECRET_GOVERNMENT`), которые не должны отображаться в обычном каталоге.
